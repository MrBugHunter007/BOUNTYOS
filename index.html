METHODHUNTING METHODOLOGY
⬡ PHASE 1 — PASSIVE RECON NO FOOTPRINT
  • [01]
    Scope Analysis
    Read program policy carefully. Note in-scope domains, excluded endpoints, reward tiers, and rules of engagement.
  • [02]
    Subdomain Enumeration
    Run subfinder, assetfinder, amass. Cross-reference crt.sh certificate transparency. Target 200–500+ unique subdomains.
  • [03]
    GitHub Dorking
    Search "target.com" api_key / password / secret on GitHub. Look for hardcoded secrets, database creds, private keys.
  • [04]
    Shodan / FOFA Recon
    Query ssl.cert.subject.cn:target.com — find exposed admin panels, S3 buckets, Jenkins, Elasticsearch.
  • [05]
    Postman / API Docs
    site:documenter.getpostman.com "target.com" — find leaked API collections with undocumented endpoints.
⬡ PHASE 2 — ACTIVE RECON PROBE
  • [01]
    Live Host Probing
    cat subs.txt | httpx -mc 200,403 -tech-detect — filter alive hosts and detect tech stack.
  • [02]
    Wayback URL Extraction
    cat subs.txt | waybackurls | httpx -mc 200 | tee wb.txt — find historical forgotten endpoints.
  • [03]
    JS File Analysis
    cat wb.txt | grep .js | tee js.txt then nuclei -l js.txt -t exposures/ to find API keys in bundles.
  • [04]
    Parameter Discovery
    Use gau, paramspider, arjun to find hidden parameters. Filter all URLs containing = signs.
  • [05]
    Tech Fingerprinting
    Wappalyzer + WhatWeb. Identify CMS versions and frameworks. Map to known CVEs.
⬡ PHASE 3 — VULNERABILITY HUNTING EXPLOIT
  • [01]
    Authentication Testing
    Test 2FA bypass via response manipulation. Check OTP brute force, rate limiting bypass via header variation.
  • [02]
    Authorization / IDOR
    Replace user IDs in API calls. Test horizontal and vertical privilege escalation on all object references.
  • [03]
    Injection Testing
    Test SQLi, XSS, SSTI, command injection. Use Burp Intruder systematically on all user-controlled parameters.
  • [04]
    SSRF Testing
    Find url=, proxy=, redirect=, host=, img= parameters. Probe 127.0.0.1, 169.254.169.254 (AWS metadata).
  • [05]
    Business Logic
    Test price manipulation, coupon abuse, race conditions, unauthorized state transitions.
⬡ PHASE 4 — REPORTING DISCLOSE
  • [01]
    Impact Assessment
    Document real-world impact clearly. Don't overclaim. Defensible Medium beats inflated Critical. Calculate CVSS honestly.
  • [02]
    PoC Creation
    Full Burp Suite request/response capture. Screen recording or screenshots proving exploitation.
  • [03]
    Report Writing
    Title, description, steps to reproduce, impact, CVSS, CWE, mitigation. Natural human prose — no overclaiming.
  • [04]
    Responsible Disclosure
    Submit via Bugcrowd/HackerOne (handle: Honor). Give vendor 90 days before disclosure.

⚡ 2FA/MFA BYPASS PLAYBOOK
// Method 1: Response Manipulation
  1. Login with valid creds, reach 2FA page
  2. Enter any wrong OTP and submit
  3. In Burp: Right-click → Do Intercept → Response to this request
  4. Change HTTP status to 200 OK
  5. Replace body with success JSON (see Payloads tab)
  6. Forward — check if session is granted
// Method 2: OTP Reuse / Brute Force
  1. Capture valid OTP request in Burp
  2. Resend same OTP immediately — check if reuse allowed
  3. Test values: null, empty, 000000, 123456
  4. Send to Intruder, brute-force 6-digit range if no rate limit
  5. Vary Accept-Language per request to bypass rate limiting
  6. Test: email+1@domain.com for registration-based limits
